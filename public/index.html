Tarot Reader

To continue practicing building servers, MySQL databases, and React skills, I decided to create an app for fun based on my newfound interest in tarot.

  • Goal: Build the simplest tarot reader possible, paring down every feature to its most basic function but using intuitive and user-centric features to ensure a delightful user experience.
  • Important Lesson I Learned: Occam’s Razor, or, Don’t Overcomplicate Things. Feeling least confident with my database abilities, I began the project focused on how I would store image and text content to be rendered dynamically on the client UI. After trials and errors taught me that storing images in MySQL wasn’t optimal, and that AWS overcomplicated things, it dawned on me that the most basic solution was the best: to optimize my images for web and store the files locally. My main takeaway: I should consider all options, technologies, and steps before starting a web project (read: plan better).
  • New Technologies I Used: React Bootstrap library features Loading Overlay and Spinner; ternary operator for rendering images.
