Shape Game Details
  • Goal: Build a shape-drawing app with my main focus being on the front-end design.
  • Important Lesson I Learned: How even a “simple app” can be rather complex. This project helped solidify the concept of JS class constructors for me, and I finally understood that constructors are necessary to instantiate new objects (with necessary properties) created by a class. In addition, I learned how important my junior high math skills can be for coding: they helped me resolve two issues.

    First, how do I draw a triangle? (It was proving to be incredibly tricky). I researched and attempted different calculations to draw triangles but the solution involved thinking totally differently: what could provide the appearance of a triangle when it wasn't actually a triangle? An empty square, of course! I drew an empty square shape, made the border thick and beveled, and colored one piece of a corner to create a triangle. (My brain exploded a bit in the process.)

    Second, how do I keep shapes from drawing outside of the canvas? For this, I limited the amount of pixels that each shape could contain in order to keep its drawn position within the canvas’s perimeter.
  • New Technologies/Skills Used: Object Oriented Programming and inheritance; lots of math

Typing Game Details
  • Goal: I was very excited to get this game working––I was a Mavis Beacon ace as a kid––but I knew the mechanics of it (i.e. the jQuery) would require extra attention. My primary aim was to make the game function well, have a good UI even if it wasn’t the prettiest, and to add game instructions to improve the UX.
  • Important Lesson I Learned: How to think a bit outside of the box in jQuery as a novice developer. This game required using keyCodes and event.which in order for the app to identify which keys were being pressed during gameplay; therefore, I had to learn that unfamiliar ways of selecting elements, such as $(‘#’ + e.which), were valid and really useful.

    After that, my last challenge was placing the yellow block/highlighter directly over the correct letter that was to be pressed on the keyboard. I made countless tiny pixel tweaks to position it correctly, but it didn’t position well over each letter until I finally used the addition assignment operator (+=) in its CSS.
  • New Technologies/Skills Used: JS keyCodes, keypress events, and event.which; bootstrap modals
